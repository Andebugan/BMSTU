Курсоры
    Способ построчной обработки выборки
    Все, что можно написать с помощью курсора, можно написать SQL
    
    Классификация:
    По области видимости:
        A) Локальные курсоры (LOCAL). Неявно освобождается.
        Б) Глобальные курсоры (GLOBAL). Мы следим ща его освобождением.
    
    По типу:
        А) Статические курсоры (STATIC). Курсор создается в момент определения.
            Создается временная копия данных для использования курсора.
        Б) Динамические курсоры (DYNAMIC). Отражают все изменения данных.
            Не поддерживает ABBSOLUTE
        В) Курсоры, управляемые набором ключей (KEYSET)
            Членство или порядок не изменяются после открытия.
        Г) Быстрые послеовательные курсоры (FAST_FORWARD).
            FORWARD_ONLY, READ_ONLY.
    
    По способу перемещения:
        А) Последовательные курсоры (FORWARD_ONLY)
        Б) Курсоры прокрутки (SCROLL)
    
    По способу распараллеливания курсора.
        А) READ ONLY. Содержимое курсора можно только считывать.
        Б) SCROLL_LOCKS. При редактированнии записи вами никто другой изменения
            вносить не может. Пессимистическая блокировка.
        В) OPTIMISTIC. Отсутсвие каких бы то ни было блокировок.
            Предполагает, что даже во премя выполнения редактирования данных
            другие пользователи смогут к ним обращаться.

    DECLARE имя-курсора CURSOR
    [область-видимости-имени-курсора]
    [возможность-перемещения-по-курсору]
    [типы-курсоров]
    [опции-распараллеливания-курсоров]
    [выявление-ситуаций-с-преобразованием-типа-курсора]
    FOR <select_instruction>
    [опция-FOR-UPDATE]

    DECLARE @myVariable CURSOR

    DECLARE MyCursor CURSOR
    FOR SLECT lastName FROM
    AdventureWorks.Person.Contact

    SET @myVariable = MyCursor;
